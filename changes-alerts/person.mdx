## Overview

The Change Alerts feature lets you monitor profiles. When our system detects a change (new job, updated position, new data in the profile, etc.), Scrapin automatically sends a JSON payload to your Alert URL endpoint instantly.

<Note>
  **Near Real-Time:** Alerts are triggered when our system detects profile
  updates during periodic crawls. Expect some delay between the profile update
  and the Alerts notification.
</Note>

<Info>Each alert delivery consumes **0.50 credits**.</Info>

## Authentication

<Warning>
  **All API requests require authentication.** You must include your API key as
  a query parameter in every request.
</Warning>

Add your API key to each request using the `apikey` query parameter:

```
GET https://api.scrapin.io/v1/alerts/*endpoint*?apikey=YOUR_API_KEY
POST https://api.scrapin.io/v1/alerts/*endpoint*?apikey=YOUR_API_KEY
```

**Get your API key:**

- Visit [https://app.scrapin.io/api](https://app.scrapin.io/api) to find your API key
- Keep your API key secure and never share it publicly

**Example:**

```bash
curl "https://api.scrapin.io/v1/alerts/url?apikey=YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

## How it works

1. **Configure** your Alert URL where you want to receive notifications
2. **Subscribe** to person profiles (resourceURL) you want to monitor
3. **Receive** automatic notifications when profiles are updated
4. **Monitor** delivery logs and manage your subscriptions

<Warning>
  - Only **HTTPS URLs** are accepted for security reasons - You can configure
  **one Alert URL per workspace**.
</Warning>

## Configuration Alert URL

<Tip>
  You can also manage your Alert URL directly from the web app at
  [https://app.scrapin.io](https://app.scrapin.io/webhooks)
</Tip>

### Get Alert URL Configuration

Retrieve your current Alert URL configuration.

<Card title="GET https://api.scrapin.io/v1/alerts/url">
  Returns the Alert URL and status for your workspace.
</Card>

**Response Example:**

```json
{
  "success": true,
  "alert": {
    "id": "uuid",
    "url": "https://your-domain.com/alert",
    "status": "active",
    "createdAt": "2025-10-14T10:00:00Z",
    "updatedAt": "2025-10-14T10:00:00Z"
  }
}
```

### Create or Update Alert URL

Configure the Alert URL where notifications will be sent.

<Card title="POST https://api.scrapin.io/v1/alerts/url">
  Create or update your Alert URL configuration.
</Card>

**Request Body:**

```json
{
  "url": "https://your-domain.com/alert",
  "status": "active"
}
```

**Parameters:**

- `url` (string, required): Your HTTPS Alert endpoint
- `status` (string, optional): `active` or `inactive` (default: `active`)

**Response:**

```json
{
  "success": true,
  "alert": {
    "id": "uuid",
    "url": "https://your-domain.com/alert",
    "status": "active",
    "createdAt": "2025-10-14T10:00:00Z",
    "updatedAt": "2025-10-14T10:00:00Z"
  }
}
```

<Warning>The URL must use HTTPS protocol. HTTP URLs will be rejected.</Warning>

## Subscriptions

<Tip>
  You can also manage your Change Alerts subscriptions directly from the web app
  at [https://app.scrapin.io](https://app.scrapin.io/webhooks)
</Tip>

### List Subscriptions

Get all your Change Alerts subscriptions with filtering and pagination.

<Card title="GET https://api.scrapin.io/v1/alerts/subscriptions">
  Retrieve all subscriptions for your workspace.
</Card>

**Query Parameters:**

- `status` (string, optional): Filter by status (`active`, `inactive`)
- `resourceType` (string, optional): Filter by resource type (`person`)
- `page` (number, optional): Page number for pagination (default: 1)
- `limit` (number, optional): Results per page (default: 50)

**Response Example:**

```json
{
  "success": true,
  "pagination": {
    "currentPage": 1,
    "pageNumber": 1,
    "perPage": 50,
    "total": 1
  },
  "data": [
    {
      "id": "uuid",
      "resourceType": "person",
      "resourceIdentifier": "john-doe",
      "resourceUrl": "https://www.social.com/in/john-doe/",
      "status": "active",
      "alertUrl": "https://your-domain.com/alert",
      "alertStatus": "active",
      "createdAt": "2025-10-14T10:00:00Z",
      "updatedAt": "2025-10-15T08:30:00Z"
    }
  ]
}
```

### Subscribe to a Profile

Subscribe to a person profile to receive updates.

<Card title="POST https://api.scrapin.io/v1/alerts/subscribe">
  Create a new subscription to a person profile.
</Card>

**Request Body:**

```json
{
  "resourceUrl": "https://www.social.com/in/john-doe/"
}
```

**Parameters:**

- `resourceUrl` (string, required): Resource URL

**Response:**

```json
{
  "success": true,
  "subscription": {
    "id": "uuid",
    "resourceIdentifier": "john-doe",
    "resourceType": "person",
    "resourceUrl": "https://www.social.com/in/john-doe/",
    "createdAt": "2025-10-14T10:00:00Z"
  }
}
```

<Note>Only one active subscription per profile per workspace is allowed.</Note>

### Unsubscribe from a Profile

Cancel a subscription to stop receiving updates.

<Card title="POST https://api.scrapin.io/v1/alerts/unsubscribe">
  Deactivate a subscription.
</Card>

**Request Body:**

```json
{
  "resourceUrl": "https://www.social.com/in/john-doe/"
}
```

**Parameters:**

- `resourceUrl` (string, required): Resource URL

**Response:**

```json
{
  "success": true,
  "subscription": {
    "id": "uuid",
    "resourceIdentifier": "john-doe",
    "resourceUrl": "https://www.social.com/in/john-doe/",
    "status": "inactive",
    "updatedAt": "2025-10-14T10:00:00Z"
  }
}
```

<Info>
  Unsubscribing sets the status to `inactive` but keeps the subscription
  history. You can reactivate by subscribing again to the same resource.
</Info>
